Case Study

Bridgewell Health Service

Building an accessible, zero-dependency website for a psychiatric healthcare facility

Client
Bridgewell Health Service
Industry
Healthcare / Psychiatric Services
Services
Web Design, Development, Accessibility
Year
2026

The Challenge

Build a professional healthcare website with strict accessibility requirements, instant load times, and zero external dependencies for maximum reliability. The site needed to serve patients, families, and staff while meeting WCAG AA compliance standards and working flawlessly on all devices and browsers.

Our Process

1

Discovery & Research

Client interviews, competitive analysis, accessibility audit of existing site, user research with patients and staff. Identified key user needs and technical requirements.

2

Design System

Created brand palette (purple #A877FE, aqua #5AEAFE), established CSS custom properties for colors, spacing, and typography. Built component library with accessibility-first patterns.

3

Development

Pure HTML5/CSS3/ES6+ JavaScript. Zero frameworks, zero build tools, zero npm packages. Progressive enhancement, mobile-first responsive design, semantic HTML for accessibility.

4

Testing & Launch

WCAG AA compliance testing with screen readers (NVDA, JAWS, VoiceOver), cross-browser QA, performance optimization, security audit, final deployment.

Technology Stack

  • Frontend: HTML5, CSS3 (custom properties, backdrop-filter, grid, flexbox, animations)
  • JavaScript: ES6+ Vanilla JS, IntersectionObserver, requestAnimationFrame, passive event listeners
  • Accessibility: ARIA labels, semantic HTML, keyboard navigation, skip links, reduced motion support
  • Performance: Lazy loading, GPU acceleration, no build tools, instant page loads
  • Design System: CSS variables, mobile-first responsive, component-based architecture
